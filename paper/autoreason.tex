\section*{Abstract}

We introduce autoreason, a method for iterative LLM refinement in subjective domains where no objective metric exists. The method constructs a subjective fitness function through independent blind evaluation, extending the autoresearch paradigm. Autoreason uses fresh isolated agents per role, blind judge panels with Borda count aggregation, and conservative tiebreaking. Across 5 task types, autoreason ranked first or second in every 7-judge blind comparison, winning 3 of 5 tasks outright. Monte Carlo analysis (5 independent runs) shows 80\% convergence rate with consistent output quality despite variable convergence speed. We additionally ran autoreason on the writing of this paper, discovering that ground-truth context for the critic eliminates hallucination. We find that the method's advantage is task-dependent: autoreason excels on tasks with genuine tradeoffs and stakeholder tensions, while direct critique-and-revise can outperform on tasks with more objectively verifiable requirements.

\section{Introduction}

LLMs exhibit systematic failures when iterating on subjective work: sycophancy, overcriticism, overcompromise, authorship bias, scope drift, and context collapse. These failures persist across prompting strategies---even in code generation, iterative refinement leads to progressive degradation~\cite{slopcodebench}.

We present autoreason, extending Karpathy's autoresearch~\cite{autoresearch} to subjective domains. Each iteration produces three competing versions---the incumbent (A), an adversarial revision (B), and a synthesis (AB)---evaluated by a blind judge panel. The method constructs a fitness function through the evaluation process itself.

We distinguish two modes. \textbf{v1 (quick pass)}: a single cycle through critic $\to$ B $\to$ AB $\to$ judge. Five LLM calls. \textbf{v2 (convergence loop)}: repeats with a full judge panel until the incumbent survives consecutive challenges. All experiments use v2.

\section{Method}

\textbf{Critic}: A fresh agent identifies problems in the current incumbent A. Finds problems only---no fixes. Optionally receives ground-truth data for fact-checking.

\textbf{Author B}: A fresh agent revises A based on the critique. Sees the task, A, and the critique. No drafting history.

\textbf{Synthesizer}: A fresh agent receives A and B with randomized labels. Produces AB by taking the strongest elements of each.

\textbf{Judge Panel}: 3 fresh agents rank A, B, AB via Borda count (3/2/1 points). Randomized labels and order. Conservative tiebreak: incumbent wins ties. Convergence at 2 consecutive A wins.

Every role is a fresh, isolated agent. The original task prompt anchors all evaluation.

\section{Experiments}

Five tasks: (1) go-to-market strategy, (2) notification system design, (3) remote work policy, (4) competitive positioning document, (5) incident response process. All experiments used claude-sonnet-4-20250514 (temp=0.8 authors, 0.3 judges). Each task ran autoreason to convergence plus four baselines (conservative, improve\_this, harsh\_critic, critique\_and\_revise) at 15 passes each from the same initial output, evaluated by a 7-judge blind panel.

The trajectory reveals a key tension. At pass 15, A had won twice consecutively (convergence). But 11 more passes followed with B winning 5 of them. Were those later B wins genuine improvements or noise?

We tested directly: a 7-judge panel comparing pass 15 vs pass 25 output chose pass 15 by 6--1. The later passes were churn, not improvement. This validates the convergence threshold but raises an important question about judge accuracy within the loop.

\subsection{Judge Accuracy}

The judges are accurate \textit{on average across a panel} but noisy \textit{per individual pass}. At pass 16, the same architecture that correctly identified the pass 15 output as strong nonetheless picked AB over A by one Borda point. This is not a failure of the method---it's inherent stochasticity in LLM evaluation. The convergence threshold filters this noise: requiring consecutive wins means a single noisy pass can't cause premature convergence, while sustained noise (as in passes 17--21) represents genuine ambiguity about which framing is better rather than clear improvement.

\subsection{Monte Carlo Analysis}

Five independent runs of Task 1 tested convergence reliability (Figure~\ref{fig:mc}).

\begin{figure}[H]
\centering
\includegraphics[width=\columnwidth]{fig_monte_carlo.png}
\caption{\textbf{Monte Carlo: 5 runs, same task.} 4/5 converged (80\%). Convergence speed varies widely (6--17 passes). Final word counts cluster around 1450--1660.}
\label{fig:mc}
\end{figure}

4 of 5 runs converged (80\%). Convergence pass varied: 6, 8, 14, 17. One run hit the 30-pass cap without converging. Despite this variance in path length, final word counts clustered tightly (1451--1660), suggesting the loop reaches a similar quality ceiling regardless of the path taken. The convergence threshold is a practical stopping rule, not a guarantee of optimality.

\subsection{Constrained Tasks}

We ran Task 1 with a constrained prompt (1000-word limit, 6 required sections with specific deliverables). The constraint eliminated word count oscillation entirely (all outputs 600--800 words vs 847--2500 unconstrained). However, it made convergence harder---B won 18 of 25 passes (72\%) because specific constraints give the critic a verifiable checklist. The loop never converged in 25 passes.

This reveals that scope constraints and convergence are orthogonal problems. Constraints stabilize output length but enable more targeted critique.

\section{Writing This Paper}

We ran autoreason on this paper (Claude Opus 4, 3 Opus judges, ground-truth context for all roles). Converged in 9 passes.

\textbf{Ground-truth critic.} Without ground-truth access, the initial generation hallucinated a fabricated ablation study, fake confidence intervals, wrong model names, and incorrect role descriptions. With ground-truth access (actual experiment data), the critic caught all four on pass 1. This suggests a general pattern: the author writes from understanding, the critic verifies against evidence---mirroring peer review.

\textbf{Judge panel integrity.} An earlier run with a broken Gemini parser reduced the effective panel to 2 judges, preventing convergence for 11 passes. Once fixed to 3 working judges, the same incumbent converged in 2 passes. Panel integrity is a prerequisite for convergence.

\section{Related Work}

\textbf{Autoresearch}~\cite{autoresearch} uses val\_bpb as an objective fitness function. Autoreason extends this to subjective domains. \textbf{SlopCodeBench}~\cite{slopcodebench} shows iterative code degradation that prompting can't fix. \textbf{ACE}~\cite{ace} documents context collapse in iterative rewriting. \textbf{LLM Council}~\cite{llmcouncil} uses multi-model panels for single-turn evaluation; autoreason adds iteration with role separation.

\section{Discussion}

\textbf{Task-dependent advantage.} Autoreason is not universally superior. It excels where genuine tradeoffs exist between competing valid perspectives---strategy, policy, process design. For tasks with concrete, verifiable requirements, simpler methods can suffice. The recommendation: use autoreason when you don't know which framing will produce the best result; use critique-and-revise when the improvement direction is clear.

\textbf{Convergence as practical stopping rule.} The threshold does not guarantee optimality. It filters noise---a single stochastic judge panel can't cause premature stopping, and post-convergence passes on Task 1 produced no improvement (pass 15 beat pass 25, 6--1). But the Monte Carlo results show 20\% non-convergence at 30 passes, meaning some runs need a cap.

\textbf{Bloat/prune oscillation} signals underdetermined scope. Constraining scope eliminates this but makes convergence harder by giving the critic concrete criteria to enforce.

\textbf{Ground-truth context} eliminates hallucination when reference material exists. The pattern generalizes beyond paper writing to any domain with source documents.

\textbf{Limitations.} All results from one model family (Anthropic). No human evaluation. No ablation studies isolating individual components. Judge accuracy within the loop is noisy per-pass.

\section{Conclusion}

Autoreason demonstrates that systematic isolation and blind evaluation can construct useful fitness functions for subjective domains. Across 5 tasks, it ranked first or second in every comparison. The method's advantage is clearest on tasks with genuine ambiguity---where the A/B/AB structure surfaces perspectives that a single revision loop cannot explore. The primary insight: the obstacle to LLM self-improvement in subjective domains is not capability but evaluation methodology.
