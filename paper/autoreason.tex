\section*{Abstract}

We introduce autoreason, a method for iterative LLM refinement in subjective domains where no objective metric exists. Building on the autoresearch paradigm, autoreason addresses fundamental limitations of LLMs when iterating on subjective work: sycophancy, overcriticism, overcompromise, authorship bias, scope drift, and context collapse. Our approach uses fresh isolated agents per iteration, blind evaluation panels with ranked choice voting and Borda count aggregation, and conservative tiebreaking. In experiments on go-to-market strategy generation, autoreason achieved unanimous first-place preference (7/7 judges, 35/35 Borda) in a 5-way comparison against four baseline methods. We additionally ran autoreason on the writing of this paper itself, converging in 9 passes with ground-truth fact-checking.

\section{Introduction}

Large language models exhibit systematic failures when iterating on subjective work. Given a document and instructions to improve it, LLMs demonstrate sycophancy (strengthening existing arguments regardless of merit), overcriticism (finding flaws where none exist), overcompromise (attempting to satisfy contradictory feedback), authorship bias (preferring their own outputs), scope drift (expanding beyond original intent), and context collapse (losing track of quality through extended interaction).

These failures persist despite sophisticated prompting strategies. Recent work demonstrates that even in objective domains like code generation, iterative refinement with critique leads to degradation rather than improvement~\cite{slopcodebench}. The problem is more acute in subjective domains where no ground truth exists to anchor evaluation.

We present autoreason, extending Karpathy's autoresearch paradigm~\cite{autoresearch} to domains requiring subjective judgment rather than objective metrics. By treating each iteration as a fresh competition between three framings---conservative (A), adversarial (B), and synthetic (AB)---with blind evaluation, autoreason constructs a subjective fitness function through the evaluation process itself.

\section{Method}

Autoreason implements a three-way competition each iteration:

\textbf{Critic Phase}: A fresh LLM agent receives the current best document (A) and task description, then identifies problems. Each critic is a new API call with no conversation history. The critic finds problems only---it does not suggest fixes.

\textbf{Response Phase}: Two separate fresh agents produce alternatives:
\begin{itemize}
\item \textbf{B (Adversarial revision)}: Receives the task, incumbent A, and the critique. Revises A to address valid criticisms.
\item \textbf{AB (Synthesis)}: Receives the task and both A and B with randomized labels. Keeps the strongest elements of each.
\end{itemize}

\textbf{Judge Phase}: A panel of 3 fresh judge agents performs blind evaluation. Each judge receives all three documents (A, B, AB) in randomized order with randomized labels. Judges rank best to worst. Rankings aggregated via Borda count (3 for first, 2 for second, 1 for third). Conservative tiebreak: incumbent A wins ties.

Every role is a fresh, isolated agent with no shared context. The original task prompt serves as the anchor throughout---judges evaluate ``which version best accomplishes the task as described.'' The system converges when A wins 2 consecutive passes.

\section{Experiments}

We evaluated autoreason on go-to-market strategy generation for an open-source Kubernetes CLI tool (5k GitHub stars, no revenue, 3-person team). All experiments used claude-sonnet-4-20250514 (temp=0.8 authors, 0.3 judges).

\subsection{Main Experiment}

We ran 26 passes ($\sim$160 LLM calls, $\sim$65 min). Figure~\ref{fig:trajectory} shows the full trajectory. The system achieved 2 consecutive incumbent wins at passes 14--15 and 24--25. Winner distribution: A 31\%, B 19\%, AB 50\%.

\subsection{Baseline Visibility}

Autoreason vs critique-and-revise: with initial output shown as baseline, 7--0 unanimous. Without baseline, 3--2. Judges need a reference point to detect drift.

\subsection{Qualitative Improvement}

Initial output: \$49/user pricing, \$100K MRR target. After 14 passes: quantified pain (\$15K/incident $\times$ 6/yr), team pricing (\$1,499/mo), validation (50+ interviews, 75\% pilot success), unit economics (CAC \$2K, LTV \$54K).

\section{Writing This Paper with Autoreason}

We ran autoreason on the writing of this paper, providing a second experimental domain. Claude Opus 4 for all roles. 3 Opus judges. All agents received the full experimental data as ground-truth context.

\subsection{Ground-Truth Critic}

Without ground-truth access, the initial Opus generation hallucinated an ablation study, fabricated confidence intervals, reported wrong model names, and described the A/B/AB roles incorrectly (Figure~\ref{fig:hallucination}). With ground-truth access, the critic caught all of these on pass 1.

\subsection{Convergence}

The paper converged in 9 passes (54 Opus calls, $\sim$40 min). Figure~\ref{fig:paper_run} shows the trajectory.

\subsection{Judge Panel Integrity}

An earlier run with a mixed panel (Opus + Sonnet + Gemini) failed to converge after 11 passes. Gemini's ranking format failed our parser, reducing the effective panel to 2 judges (Figure~\ref{fig:judge_integrity}). With 2 judges, Borda scores compressed and convergence became impossible. Once fixed to 3 Opus judges, the same incumbent converged in 2 passes.

\section{Related Work}

\textbf{Autoresearch}~\cite{autoresearch} uses val\_bpb as an objective fitness function. Autoreason extends this to subjective domains. \textbf{SlopCodeBench}~\cite{slopcodebench} shows agents produce progressively worse code each iteration---prompting doesn't fix it. \textbf{ACE}~\cite{ace} documents context collapse in iterative rewriting. \textbf{LLM Council}~\cite{llmcouncil} uses multi-model panels for single-turn evaluation; autoreason adds iteration with role separation.

\section{Discussion}

\textbf{Bloat/prune oscillation} signals underdetermined tasks. AB adds complexity, B prunes it. Word counts: 847 $\to$ 1800 $\to$ 1644 $\to$ 1758 over 26 passes.

\textbf{Conservative beats naive iteration.} Doing almost nothing outperformed 15 passes of ``improve this,'' ``harsh critic,'' and ``critique and revise.'' Structural protection is necessary for iterative refinement.

\textbf{Ground-truth context} eliminates hallucination. The author writes from understanding; the critic verifies against evidence---mirroring peer review.

\textbf{Beyond subjective domains.} Code can pass tests while degrading~\cite{slopcodebench}. Autoreason could complement objective metrics with quality evaluation.

\textbf{Limitations.} Single task domain for main experiment. No statistical significance testing. No ablations. All results from Anthropic models.

\section{Conclusion}

Autoreason demonstrates that systematic isolation and blind evaluation can construct effective fitness functions for subjective domains. The method achieved unanimous preference over four baselines and wrote a version of this paper through 9 passes of self-refinement. The primary obstacle to LLM self-improvement in subjective domains is not capability but evaluation methodology.
